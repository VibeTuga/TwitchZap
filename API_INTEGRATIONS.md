# TwitchZap — API Integrations

## 1. Twitch Helix API

The primary external API. Used to verify streams, fetch metadata, and authenticate users.

### Authentication

TwitchZap uses two auth flows:

**App Access Token** (server-to-server, for stream lookups):
```
POST https://id.twitch.tv/oauth2/token
  ?client_id={TWITCH_CLIENT_ID}
  &client_secret={TWITCH_CLIENT_SECRET}
  &grant_type=client_credentials
```

**User Access Token** (OAuth, for user login):
```
GET https://id.twitch.tv/oauth2/authorize
  ?client_id={TWITCH_CLIENT_ID}
  &redirect_uri={APP_URL}/api/twitch/callback
  &response_type=code
  &scope=user:read:email
```

### Endpoints Used

#### Get Stream Info (Verify Live Status)
```
GET https://api.twitch.tv/helix/streams?user_login={username}
Headers: 
  Authorization: Bearer {app_access_token}
  Client-Id: {TWITCH_CLIENT_ID}

Response (if live):
{
  "data": [{
    "id": "stream_id",
    "user_id": "12345",
    "user_login": "coolstreamer",
    "user_name": "CoolStreamer",
    "game_id": "509658",
    "game_name": "Just Chatting",
    "type": "live",
    "title": "Stream title here",
    "viewer_count": 12,
    "started_at": "2026-03-25T10:00:00Z",
    "thumbnail_url": "https://..."
  }]
}

Response (if offline):
{ "data": [] }
```

**Usage:** Called before a stream starts broadcasting on TwitchZap, and polled every 60 seconds during broadcast to detect offline status.

#### Get User Info (Channel Metadata)
```
GET https://api.twitch.tv/helix/users?login={username}
Headers: same as above

Response:
{
  "data": [{
    "id": "12345",
    "login": "coolstreamer",
    "display_name": "CoolStreamer",
    "profile_image_url": "https://...",
    "description": "I play games and stuff",
    "created_at": "2020-01-15T00:00:00Z"
  }]
}
```

**Usage:** Called when a stream is submitted to fetch channel metadata and avatar.

#### Get Game/Category Info
```
GET https://api.twitch.tv/helix/games?id={game_id}

Response:
{
  "data": [{
    "id": "509658",
    "name": "Just Chatting",
    "box_art_url": "https://..."
  }]
}
```

**Usage:** Enrich the broadcast display with category artwork.

### Twitch Embed (Client-Side)

The Twitch interactive embed is used to display the live stream in the browser:

```html
<iframe
  src="https://player.twitch.tv/?channel={username}&parent={APP_DOMAIN}"
  height="100%"
  width="100%"
  allowfullscreen>
</iframe>
```

**Important:** The `parent` parameter must match the domain serving the page. For local dev, use `localhost`.

### Rate Limits

- App Access Token requests: 800 per minute
- Helix API: 800 points per minute (most endpoints cost 1 point)
- We'll call at most ~2 endpoints per minute per broadcast (stream check + metadata), well within limits

### Error Handling

| Status | Meaning | Action |
|--------|---------|--------|
| 401 | Token expired | Refresh app access token |
| 404 | Channel not found | Reject submission, notify user |
| 429 | Rate limited | Exponential backoff, retry after header value |
| 500+ | Twitch server error | Retry with backoff, log error |

## 2. Supabase Auth (Twitch OAuth Provider)

Supabase handles the OAuth flow natively when configured with Twitch as a provider.

### Setup

In Supabase Dashboard → Authentication → Providers → Twitch:
- Client ID: `{TWITCH_CLIENT_ID}`
- Client Secret: `{TWITCH_CLIENT_SECRET}`
- Redirect URL: Auto-generated by Supabase

### Login Flow

```typescript
// Client-side
const { data, error } = await supabase.auth.signInWithOAuth({
  provider: 'twitch',
  options: {
    redirectTo: `${window.location.origin}/auth/callback`,
    scopes: 'user:read:email',
  }
});

// Callback page
const { data: { session } } = await supabase.auth.getSession();
// Session contains user info + Twitch access token
```

### User Metadata Extraction

After auth, Supabase stores Twitch metadata in `auth.users.raw_user_meta_data`:

```json
{
  "sub": "12345",
  "name": "CoolStreamer",
  "email": "user@example.com",
  "picture": "https://...",
  "preferred_username": "coolstreamer"
}
```

We use this to populate the `users` table on first login via a database trigger or API route.

## 3. Supabase Realtime

### Channel Configuration

```typescript
// Subscribe to broadcast changes
const broadcastChannel = supabase
  .channel('broadcast-live')
  .on('postgres_changes', {
    event: '*',
    schema: 'public',
    table: 'broadcasts',
    filter: 'status=eq.live'
  }, (payload) => {
    // Handle broadcast updates
  })
  .subscribe();

// Subscribe to vote counts (custom broadcast channel)
const voteChannel = supabase
  .channel('votes-live')
  .on('broadcast', { event: 'vote_update' }, (payload) => {
    // { skip_count: 12, stay_count: 8, total: 20 }
  })
  .subscribe();

// Subscribe to queue changes
const queueChannel = supabase
  .channel('queue-updates')
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',
    table: 'queue',
    filter: 'status=eq.waiting'
  }, (payload) => {
    // New submission added
  })
  .subscribe();
```

### Presence (Active Viewer Count)

```typescript
const presenceChannel = supabase.channel('viewers', {
  config: { presence: { key: userId } }
});

presenceChannel
  .on('presence', { event: 'sync' }, () => {
    const state = presenceChannel.presenceState();
    const viewerCount = Object.keys(state).length;
  })
  .subscribe(async (status) => {
    if (status === 'SUBSCRIBED') {
      await presenceChannel.track({ user_id: userId, joined_at: new Date() });
    }
  });
```

## 4. Vercel Cron

### Queue Rotation Cron

Configured in `vercel.json`:

```json
{
  "crons": [
    {
      "path": "/api/cron/rotate",
      "schedule": "* * * * *"
    }
  ]
}
```

The cron endpoint is secured with a `CRON_SECRET` environment variable:

```typescript
// /api/cron/rotate/route.ts
export async function GET(request: Request) {
  const authHeader = request.headers.get('authorization');
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return Response.json({ error: 'Unauthorized' }, { status: 401 });
  }
  
  // Rotation logic here
}
```

### Cron Responsibilities

Every minute, the cron job:
1. Checks if the current broadcast has expired
2. Tallies votes if in voting window
3. Applies skip/extend decisions
4. Transitions to next stream if needed
5. Verifies the current stream is still live on Twitch
6. Awards watch-time points to active viewer sessions
7. Cleans up stale viewer sessions (no heartbeat in 5+ minutes)

## 5. Internal API Routes

### POST /api/votes
Submit a vote on the current broadcast.

```typescript
Request:  { broadcast_id: string, vote: "skip" | "stay" }
Response: { success: true, current_totals: { skip: 12, stay: 8 } }
Errors:   401 (not authenticated), 400 (not in voting window), 409 (already voted)
```

### POST /api/submissions
Submit a stream to the queue.

```typescript
Request:  { twitch_username: string }
Response: { success: true, queue_position: 15, stream: { ... } }
Errors:   401 (not authenticated), 400 (invalid channel), 409 (in cooldown or already queued)
```

### GET /api/broadcasts/current
Get the current live broadcast.

```typescript
Response: {
  broadcast: { id, stream, status, scheduled_end_at, voting_opens_at, extensions_count, ... },
  votes: { skip: 12, stay: 8, total: 20, quorum: 5, user_vote: "stay" | null },
  viewers: 45
}
```

### GET /api/queue
Get the upcoming queue.

```typescript
Response: {
  queue: [{ position, stream, submitted_by, submitted_at }, ...],
  total: 23
}
```

### POST /api/boosts/cooldown
Spend Zap Points to reduce a stream's cooldown.

```typescript
Request:  { stream_id: string }
Response: { success: true, points_spent: 100, new_cooldown_hours: 18 }
Errors:   401 (not authenticated), 400 (insufficient points), 409 (stream not in cooldown)
```

### POST /api/liveness/report
Client reports stream offline/online status (from Twitch Embed events).

```typescript
Request:  { broadcast_id: string, status: "offline" | "online" }
Response: { success: true, consensus_count: 3, grace_period_active: true }
Errors:   401 (not authenticated), 404 (broadcast not found)
Notes:    Server tracks unique reporters. Triggers grace period at 3+ reports 
          (or fewer if low viewer count). Deduplicates per user.
```

### GET /api/liveness/status
Get current liveness state for the active broadcast.

```typescript
Response: {
  broadcast_id: string,
  is_healthy: boolean,
  offline_detected_at: string | null,
  grace_period_expires_at: string | null,
  grace_period_remaining_seconds: number | null,
  offline_reporters_count: number,
  recovery_count: number
}
```

### GET /api/leaderboard
Get community rankings.

```typescript
Query:    ?type=points|submissions|votes&period=week|month|all
Response: {
  rankings: [{ user, value, rank }, ...],
  user_rank: { rank: 42, value: 1250 }
}
```
